# M1 手工验收清单

本清单用于确认当前定义下的 **M1：桌面存在感 + 最小照料闭环** 是否完成。

## 自动化证据

- 自动化脚本：`npm run verify:m1:manual`
- 自动化报告：`docs/M1-MANUAL-QA-REPORT.md`

自动化只能证明基线功能存在，不能替代人工观察与交互确认。

## 环境

- [x] 开发环境验证通过：`npm run electron:dev`
- [x] 打包环境验证通过：`npm run pack` 后手动启动应用

## M1-A：桌面存在感

### 1. 启动自动出现

- [x] 应用启动后，猫咪无需额外交互即可出现在桌面上
- [x] 静态猫咪主视觉完整显示，没有裁切或错位

### 2. 自主漫游

- [x] 猫咪会自行移动
- [x] 猫咪会短暂停顿
- [x] 猫咪会转向后继续移动

### 3. 边界安全

- [x] 猫咪始终处于当前显示器可用工作区内
- [x] 不会出现明显越界或部分逃出屏幕的情况

### 4. 重启后恢复

- [x] 猫咪移动到非默认位置后退出应用
- [x] 重新启动后，猫咪出现在上次位置附近

## M1-B：最小照料闭环

### 5. 状态初始化

- [x] 首次启动后可看到 `Food / Water / Play / Trust` 状态
- [x] 菜单栏入口可见，并可打开照料菜单

### 6. 托盘交互

- [x] 托盘菜单可执行 `Feed`
- [x] 托盘菜单可执行 `Give water`
- [x] 托盘菜单可执行 `Pet`

### 7. 状态变化

- [x] 喂食后 `Food` 提升
- [x] 喝水后 `Water` 提升
- [x] 抚摸后 `Play` 与 `Trust` 提升

### 8. 持久化

- [x] 完成至少一次照料动作后退出应用
- [x] 重新启动后，状态变化仍然保留

## 打包门槛

- [x] `npm run verify:runtime` 通过
- [x] 打包后的应用没有回退到旧版 `index.html` 路径
