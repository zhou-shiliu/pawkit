# M3 鲜活感升级手工 QA 清单

> 状态：待 M3 实现完成后执行。  
> 目的：把“更像猫、更愿意整天挂着看”拆成可复验的人工检查项，并明确必须同时保护 M2 / M1 门禁。

## 准备

- [ ] 已执行 `npm test`
- [ ] 已执行 `npm run build`
- [ ] 已执行 `npx tsc --noEmit --project tsconfig.json`
- [ ] 已执行 `npm run verify:m3:livelike`
- [ ] 已执行 `npm run verify:m2:presence`
- [ ] 已执行 `npm run verify:m1-closure`

## 启动第一眼

- [ ] 启动后 3-8 秒内，猫至少出现一次明显不同于初始静态图的状态切换。
- [ ] 初始姿态与后续姿态可明确区分，不再是单一贴图长时间不变。
- [ ] 首屏切换看起来像“猫有状态变化”，而不是随机闪图。

## 工作态（work）

- [ ] 工作态默认进入低打扰状态集合。
- [ ] 工作态至少出现 2 个以上 visual states，但切换频率不过快。
- [ ] 工作态仍保持 docked，不横穿工作区。
- [ ] 工作态不会出现 idle-only 的大动作或过度活跃姿态。
- [ ] 工作态照料提示过滤仍符合 M2：非紧急隐藏，紧急保留。

## 闲置态（idle）

- [ ] 闲置态比工作态明显更活。
- [ ] idle `move` 相位与 `pause` 相位的 visual state 可区分。
- [ ] `walk-a / walk-b` 与 pause 态不会混用。
- [ ] 闲置态仍保留 M1 roaming 语义：`pause / turn / move` 可观察。

## 视觉稳定性

- [ ] `data-visual-state` 或等价标识可稳定暴露给自动化。
- [ ] `data-cat-pose` 或等价姿态标识存在，避免依赖图片二进制差异。
- [ ] 姿态切换时没有明显锚点漂移、裁切或闪烁。
- [ ] 左右朝向镜像与姿态切换组合时不出现错位。
- [ ] 阴影与主视觉跟随同一状态所有者，没有“影子和猫不同步”。

## 回归保护

- [ ] `verify:m2:presence` 继续通过。
- [ ] `verify:m1-closure` 继续通过。
- [ ] `cat.roaming` 仍只表示真实漫游位置，不被 dock 坐标污染。
- [ ] `mainWindow.setBounds` 仍保持单一所有者语义。

## 主观结论

- [ ] 工作态看起来“活着”，但不吵。
- [ ] 闲置态比工作态更有陪伴感，而不是更混乱。
- [ ] 整体观感已达到：**“它不再像静态图，我愿意整天开着继续观察。”**

## 备注

- 如果任一项失败，先回溯对应自动化证据（`verify:m3:livelike` / `verify:m2:presence` / `verify:m1-closure`），再决定是视觉状态机、资源锚点、还是回归门禁问题。
- 此清单用于 M3 实现验收；在实现未落地前，不应把条目误标为已完成。
