# Pawkit Pet MVP 手工 QA 清单

目标版本：`v0.1.0-mvp`  
目标分支：`codex/pet-mvp-reset-runtime`  
最后更新：2026-05-15

## QA 目标

确认 Pawkit MVP 已满足“普通用户可以试用”的最小闭环：启动能看到宠物，宠物能动、能拖、能导入/切换，并能记住上次选择。

## 测试前准备

1. 安装依赖：

   ```bash
   npm install
   ```

2. 启动应用：

   ```bash
   npm run dev
   ```

3. 准备一个 Codex Pet 风格宠物包，支持 `.zip` 或已解压目录。

## 手工 QA 项目

### 1. 首次启动

- [ ] 运行 `npm run dev` 后 Electron 正常启动。
- [ ] 桌面上出现宠物窗口。
- [ ] 默认宠物为内置 `Pawkit Sprout`，或恢复上次选择的宠物。
- [ ] 托盘菜单存在 Pawkit 菜单项。
- [ ] 默认界面没有旧照料 HUD 或复杂设置页。

期望结果：用户无需额外资产也能看到一个动态桌宠。

### 2. 拖拽移动

- [ ] 按住宠物可以拖动。
- [ ] 松开后宠物停留在新位置。
- [ ] 向左拖动时宠物显示向左奔跑动作。
- [ ] 向右拖动时宠物显示向右奔跑动作。
- [ ] 同一次拖拽中先向左再向右，方向能及时切换。
- [ ] 拖到屏幕边缘时不会完全丢出屏幕。

期望结果：拖拽自然，方向反馈即时。

### 3. 点击反馈

- [ ] 单击宠物会触发关注/挥手类反馈。
- [ ] 双击宠物会触发成功/跳跃类反馈。
- [ ] 反馈结束后能回到稳定状态。
- [ ] 头顶提示短暂出现，不常驻干扰。

期望结果：宠物有轻量回应，但不会变成通知弹窗。

### 4. 导入 ZIP 宠物包

- [ ] 从托盘点击 `导入宠物包…`。
- [ ] 导入面板显示在屏幕中间附近。
- [ ] 点击 `ZIP 包` 后系统文件选择框显示在前景。
- [ ] 选择有效 `.zip` 宠物包后导入成功。
- [ ] 导入后宠物立即切换为新宠物。
- [ ] 托盘 `切换宠物` 列表出现该宠物。

期望结果：ZIP 导入路径完整可用。

### 5. 导入目录宠物包

- [ ] 从托盘点击 `导入宠物包…`。
- [ ] 点击 `选择目录` 后系统目录选择框显示在前景。
- [ ] 选择包含 `pet.json` 与 `spritesheet.webp` 的目录后导入成功。
- [ ] 导入后宠物立即切换为新宠物。
- [ ] 托盘 `切换宠物` 列表出现该宠物。

期望结果：已解压宠物包导入路径完整可用。

### 6. 切换宠物

- [ ] 托盘菜单中进入 `切换宠物`。
- [ ] 可以看到内置宠物与已导入宠物。
- [ ] 当前宠物 radio 状态正确。
- [ ] 切换到另一个宠物后桌面宠物立即变化。
- [ ] 托盘顶部 `当前宠物：...` 文案更新。

期望结果：用户可以自由换宠物外形。

### 7. 重启记忆

- [ ] 切换到非默认宠物。
- [ ] 完全退出 Pawkit。
- [ ] 重新运行 `npm run dev`。
- [ ] 启动后仍显示上次选择的宠物。
- [ ] 宠物位置尽量恢复到上次保存位置。

期望结果：宠物选择不会每次重启回到默认宠物。

### 8. 显示、隐藏与找回

- [ ] 托盘点击 `隐藏宠物` 后桌面宠物隐藏。
- [ ] 托盘点击 `显示宠物` 后宠物重新显示。
- [ ] 将宠物拖到边缘后点击 `找回 / 重置位置`。
- [ ] 宠物回到可见、可操作的位置。

期望结果：用户不会因为位置问题“丢宠物”。

### 9. 异常宠物包

- [ ] 导入缺少 `pet.json` 的目录会失败，并显示可理解错误。
- [ ] 导入缺少 `spritesheet.webp` 的目录会失败，并显示可理解错误。
- [ ] 导入取消时不会改变当前宠物。

期望结果：失败安全，不破坏当前可用状态。
