# Pawkit v0.1.0 MVP 发布说明

发布日期：2026-05-15
发布分支：`main`
目标标签：`v0.1.0`

## 版本定位

`v0.1.0` 是 Pawkit 的 Pet MVP Reset 收口版本。这个版本不再以旧照料系统为中心，而是聚焦为一个 **Codex Pet 兼容桌面宠物陪伴应用**。

核心目标：

> 让用户快速体验一个会动、可拖拽、可替换、会记住选择的轻量桌面宠物。

## 主要能力

### Codex Pet 兼容

- 支持 `pet.json + spritesheet.webp` 宠物包。
- 支持常见 Codex Pet 动作别名归一化。
- 支持内置宠物 `Pawkit Sprout`。
- 支持社区宠物包作为本地导入资产。

### 桌面宠物播放

- 启动后显示透明桌面宠物窗口。
- 使用 spritesheet 动画播放宠物动作。
- 默认隐藏旧照料 HUD 和复杂设置。
- 缩小宠物显示尺寸，降低遮挡感。
- 非闲置状态短暂展示头顶提示。

### 交互

- 鼠标拖拽移动宠物。
- 拖拽方向映射到左右奔跑动作。
- 单击触发关注反馈。
- 双击触发成功反馈。
- 托盘支持显示、隐藏、找回/重置位置。

### 宠物管理

- 托盘支持导入 `.zip` 宠物包。
- 托盘支持导入已解压宠物目录。
- 托盘支持切换宠物。
- 重启后恢复上次选择的宠物。
- 导入资产写入用户数据目录，不写入仓库。

## 当前不包含

- AI 聊天
- 语音互动
- 多屏高级控制
- 复杂设置页
- 公开分发安装器流程
- 旧照料系统默认入口
- 更复杂的电子宠物数值养成系统

## 验证状态

自动化门禁：

- `npm test`
- `npm run verify:pet-mvp`
- `npm run verify:pet-dialog`
- `npx tsc --noEmit --project tsconfig.json`
- `npm run build`
- `git diff --check`

手工 QA：

- 清单见 `docs/releases/v0.1.0/QA.md`
- 发布前需要完成手工 QA 结论填写

## 已知限制

- 当前安装包未做正式代码签名 / macOS 公证，首次打开可能出现系统安全提示。
- 宠物包格式以 Codex Pet 风格为主，非标准包可能需要整理后导入。
- 头顶提示是轻量固定文案，暂不支持自定义。
- 没有复杂设置页，行为参数主要由代码内 MVP 配置控制。
- legacy 照料能力仍留在代码中，但不作为当前默认体验。

## 发布流程

1. 在 `main` 完成文档、验证和手工 QA。
2. 在 `main` 打标签：

   ```bash
   git tag v0.1.0
   ```

3. 推送分支和标签：

   ```bash
   git push origin main
   git push origin v0.1.0
   ```

## 下一阶段建议

MVP 发布后，下一阶段不要立刻扩复杂系统。建议优先观察：

- 用户是否能顺利导入社区宠物包。
- 用户是否理解托盘里的切换和找回操作。
- 宠物尺寸与头顶提示是否舒适。
- 拖拽、位置恢复、重启记忆是否稳定。

如果反馈稳定，再考虑 v0.2.0：宠物包管理体验、更多内置宠物、状态提示文案优化、轻量偏好设置。
