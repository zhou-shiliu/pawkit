# Project Mew - 桌面猫咪数字宠物

> 核心理念：尽量真实还原真实猫咪习性，关系随时间建立，像养一只真正的猫。

---

## 项目概述

### 最新方向更新

当前构建已经进入双模式存在感：

- 工作态：猫咪贴在屏幕边缘静止，不横穿工作区，只保留轻微微动，并且只在紧急需求时提醒
- 闲置态：用户无操作达到一定时长后，猫咪恢复自由活动
- 闲置阈值：用户可在 `10 分钟 / 30 分钟 / 1 小时` 三个预设中选择，默认 `10 分钟`

这条方向约束后续阶段：先调优陪伴感与提示边界，不立即引入复杂设置页、声音、语音、sprite 或 IK。

### 是什么
一个运行在电脑桌面上的数字猫咪，有自己的情绪、需求和行为逻辑，会对主人的互动产生真实的反应。

### 核心理念
- **MVP 先验证照料闭环**：优先使用单张静态主视觉 + 头顶需求提示 + 菜单栏照料入口，先确认陪伴感与照料反馈是否成立
- **关系需要培养**：信任等级需要 2-4 周的正向互动才能达到最高
- **有脾气的生命**：不是讨好人的宠物，有自己的情绪和意愿
- **成长性**：体重、性格随养育方式变化

### 技术栈
- Electron（桌面应用框架）
- React 提示层 + 静态插画主视觉
- 桌面窗口位移驱动“存在感”
- 轻量状态驱动 UI（非 sprite、非 IK）
- 状态持久化（本地存储）

### 1. 视觉 & 渲染

**分辨率 & 窗口**
- 全屏活动边界（覆盖整个桌面）
- 猫只能在屏幕内活动
- 窗口置顶（`always on top`）
- 无需障碍物寻路，直线移动

**美术风格**
- 温暖的静态猫咪主视觉
- 清晰、低噪声、适合小窗口常驻
- 优先保证轮廓识别和状态信息可读性

**当前 MVP 视觉方案**
- 使用 1 张猫咪静态主视觉承载桌面角色
- 可根据漫游方向做左右翻转
- 猫咪只在有需求时显示头顶提示
- 用户通过菜单栏入口完成 `喂食 / 加水 / 陪玩`
- 不在 M1 中引入 sprite sheet、骨骼 IK 或多动作动画系统

### 2. 动机系统（核心变量）

| 变量 | 范围 | 说明 |
|------|------|------|
| 能量 | 0-100 | 低于20拒绝玩耍，精力充沛更活跃 |
| 饱腹感 | 0-100 | 0=非常饿，100=很饱 |
| 饥饿度 | 0-100 | 0=不饿，100=极度饿（计算：100-饱腹感） |
| 压力 | 0-100 | 越低越好，影响睡眠质量和食欲 |
| 气味标记 | 0-100 | 随时间衰减，正向互动恢复，代表"孤独感" |
| 信任度 | 1-5 | 等级越高，猫越放松、互动越多 |
| 猎物欲望 | 0-100 | 累积到一定程度触发狩猎行为 |
| 疲劳度 | 0-100 | 清醒时间越长越高，>70强制睡眠 |
| 体重 | kg | 根据进食历史成长变化 |

### 3. 孤独感系统

**模型：气味标记衰减（非人类式孤独）**

- 猫通过蹭、磨爪、喷尿（数字版：活动轨迹）标记领地
- 这些标记随时间自然衰减
- 主人互动 → 猫在主人附近"气味标记" → 衰减变慢

**衰减规则**
- 无主人在场（鼠标无活动 > 30分钟）：气味标记 -1/10分钟
- 主人在场（鼠标有活动）：气味标记 +1/30分钟（仅在猫附近时）
- 气味标记 < 30：猫感到孤独，行为改变

**孤独时的表现**
- 更频繁的踱步、叫声
- 更强的猎物欲望（发泄）
- 对任何输入反应更强烈
- 长期孤独 → 压力上升

### 4. 喂食系统

**定量供给 + 自主进食**
- 主人设置每日猫粮总量（单位：克）
- 每日凌晨00:00手动重置（用户按悬浮球"重置食盆"）
- 猫根据饥饿度自主决定何时进食
- 食盆猫粮吃完 → 提醒主人（悬浮球图标变化 + 气泡提示）

**每日食量计算**
```
每日热量需求 = 体重(kg) × 40~50 kcal
例：4kg猫 → 160~200 kcal/天

食物配比：70%干粮 + 30%湿粮
```

**体重成长系统**
- 持续吃多 → 体重缓慢增加（上限8kg）
- 持续少吃 → 体重缓慢下降（下限2kg）
- 体重变化影响每日热量需求（动态调整）

**进食行为规则**
- 猫在休息/做其他事 → 检测饥饿度

| 饥饿度 | 行为 |
|--------|------|
| > 90 | 焦躁，发出饥饿叫声，主动找主人求食物 |
| 70-90 | 立即放下当前行为，去找食物（优先级高于警惕） |
| 40-70 | 注意食物但不立刻去；信任≥3在人附近吃，<3等人离开 |
| < 40 | 知道有食物但不急，可能先做其他事 |

**进食被打断（4级严重度）**

| 等级 | 触发条件 | 反应 |
|------|---------|------|
| 轻度 | 轻微声响/动静 | 抬头看一眼，继续吃 |
| 中度 | 主人靠近 | 警惕，稍停，但不跑 |
| 重度 | 突然干扰/被抓 | 叼着食物换地方 |
| 极度 | 严重惊吓 | 当天停止进食，压力+30 |

**食物过期**
- 每日00:00重置时，未吃完的食物消失（算浪费）
- 下次进食从新的一天的配额开始

**水碗**
- 无限量，自动续满
- 不影响饱腹感（独立系统）

### 5. 信任等级系统

**5级信任模型**

| 等级 | 名称 | 表现 | 到达条件 |
|------|------|------|---------|
| 1 | 陌生/警惕 | 保持距离，不主动接近，被触碰可能逃跑 | 初始状态 |
| 2 | 试探 | 可以近距离观察，允许轻触，反应谨慎 | 3-5天正向互动 |
| 3 | 熟悉 | 在主人附近活动，不躲避，允许抚摸 | 1-2周正向互动 |
| 4 | 信任 | 主动蹭腿求摸，露出腹部睡觉（偶尔） | 2-3周正向互动 |
| 5 | 依恋 | 形影不离，高压时找主人庇护 | 3-4周正向互动 |

**信任度变化规则**
- 正向互动（喂食、轻摸、说话、玩耍成功）：信任+1（每日上限+2）
- 强迫互动（强行抱、长时间禁锢）：信任-1
- 长期高压（>70持续3天以上）：信任-1
- 信任降到1后，猫表现如同新猫（关系可重建）

**信任影响所有行为**
- 信任≥3：主人在场不构成压力，可靠近食物
- 信任≥4：主动寻求互动，允许更多接触
- 信任<2：所有接近都是威胁，保持最大距离

### 6. 召唤机制

**悬浮球（主要召唤方式）**

| 操作 | 效果 |
|------|------|
| 点击（短按） | 猫看向主人方向，播放"喵？"，进入待命状态 |
| 长按（2秒） | 打开快捷菜单 |
| 猫不在附近时点击 | 猫被召唤，向屏幕中央移动 |
| 猫在附近时点击 | 猫轻轻蹭一下悬浮球（玩耍） |

**快捷菜单（长按弹出）**
- 喂食（直接给粮）
- 玩耍（激活激光点）
- 摸摸（抚摸模式）
- 设置（打开设置面板）

**待命状态**
- 召唤后猫进入待命状态
- 等待主人的下一步指令（鼠标/输入/说话）
- 10秒无互动 → 猫退出待命，恢复自然行为

**Phase 2 升级**
- 语音唤醒词（如"小咪"）
- 猫听到唤醒词后主动过来

### 8. 社交/互动系统

**鼠标 = 主人的手**

| 鼠标动作 | 猫的反应（信任等级≥2） |
|---------|---------------------|
| 鼠标在猫附近移动 | 猫注意到，开始关注 |
| 鼠标悬停在猫身上 | 轻微不安，等待 |
| 单击猫 | 互动（信任等级决定反应激烈程度） |
| 双击桌面空白处 | 猫抬头张望，确认位置 |
| 鼠标快速移动 | 猫视为猎物（激光点模式） |
| 鼠标静止不动 | 非威胁，猫放松 |

**点击互动（信任等级影响）**

| 等级 | 反应 |
|------|------|
| 1 | 逃跑/躲开 |
| 2 | 轻微后退，不跑，轻声叫 |
| 3 | 蹭一下鼠标，轻声叫 |
| 4 | 主动凑近，呼噜，蹭 |
| 5 | 翻肚皮打滚（偶尔），强烈撒娇 |

**抚摸系统（简化版）**
- 点击悬浮球"摸摸" → 进入抚摸模式
- 鼠标在猫身上移动 → 猫发出呼噜声
- 抚摸时间过长（> 30秒）→ 猫跑开（简化版，不做过度刺激细节）
- 抚摸让猫放松：压力-15/次，能量+8

**玩耍系统**
- 鼠标快速移动 → 猫进入狩猎模式（激光点）
- 猫跟踪、扑击光点
- 能量 -10/次，猎物欲望 -30/次
- 玩耍后猫可能舔毛整理

**猫主动互动**
- 随机触发（不固定频率）
- 触发条件：信任等级、当前情绪、状态

| 信任等级 | 可能的主动行为 |
|---------|--------------|
| ≥2 | 蹭腿 |
| ≥3 | 头蹭、手蹭 |
| ≥4 | 翻肚皮（偶尔） |
| ≥5 | 跟在身后走来走去 |

### 9. 狩猎/猎物系统

**猎物欲望累积**
- 清醒状态下：+1/5分钟
- 看到窗外鸟/虫子（环境感知）：+15/次
- 听到尖锐声响：+5/次
- 其他猫的狩猎行为：+10/次

**触发狩猎行为**
```
猎物欲望 > 70 + 能量 > 40 + 猫处于活跃状态
→ 猫开始在屏幕内寻找移动目标
```

**扑击流程**
1. 猫发现光点 → 瞳孔放大，耳朵转向
2. 猫跟踪光点 → 潜行姿态移动
3. 光点移动 → 猫扑击（动画）
4. 光点停止 → 猫等待/犹豫
5. 持续不动 → 猫失去兴趣，猎物欲望 -10

**扑击消耗**
- 能量 -15
- 猎物欲望 -40
- 成功 → 满足感 +20
- 失败 → 猎物欲望 -20，舔毛自我安抚

### 10. 安全/压力系统

**压力源**

| 来源 | 压力值 |
|------|--------|
| 长时间孤独（气味标记衰减后） | +1/小时 |
| 进食被打断（严重干扰） | +30/次 |
| 持续被盯着 | +5/分钟 |
| 强迫互动（抓住/抱起太久） | +10/次 |
| 环境突然变化 | +10/次 |
| 猎物欲望长期无法满足（>24小时） | +20 |

**压力缓解**

| 行为 | 效果 |
|------|------|
| 信任的人在身边（不强迫互动） | -5/30分钟 |
| 猫主动的抚摸互动 | -15/次 |
| 成功狩猎/玩耍 | -20/次 |
| 深睡眠 | -10/次 |
| 在安全区域梳理毛发 | -5/次 |

**高压状态表现**

| 压力值 | 表现 |
|--------|------|
| > 70 | 拒绝进食，躲藏行为，飞机耳，尾巴向下/夹紧，不主动接近主人 |
| > 90 | 完全不回应，身体蜷缩，过度舔毛（可能脱毛），食欲完全丧失 |

**干预机制**
主人发现高压信号 → 停止强迫互动 → 给猫空间 → 压力自然下降

**安全信号**
- 信任的人在身边 = 安全感来源
- 信任≥3：主人正常走动不造成压力，压力恢复更快
- 信任<2：任何接近动作都是威胁，猫保持距离

### 11. 能量/休息系统

**能量值（0-100）**

| 消耗来源 | 数值 |
|---------|------|
| 四处走动 | -1/分钟 |
| 玩耍（激光点） | -10/次 |
| 进食消化 | -3/次 |
| 应激状态 | -2/小时 |

| 恢复来源 | 数值 |
|---------|------|
| 浅睡眠 | +5/15分钟 |
| 深睡眠 | +15/15分钟 |
| 放松抚摸 | +8/次 |
| 饱腹感充足 | +3/次 |

| 能量临界值 | 行为 |
|-----------|------|
| < 20 | 非常疲倦，优先寻找休息地点，拒绝玩耍 |
| < 40 | 开始降低活动量 |
| > 80 | 精力充沛，更活跃 |

**活动状态机**

```
睡眠(深/浅) ◄────── 睡醒/能量恢复
   │                        │
   │ 睡醒                   │ 饥饿度↑ / 发现猎物
   ▼                        ▼
休息(打盹) ────── 应激/被惊吓 ────── 活跃(游荡)
   │                                          ▲
   │                                          │
   └────────── 触发压力源 ────────────────────┘
```

**睡眠质量**
- 疲劳度（0-100）：清醒时间越长越高，>70强制睡眠
- 压力高时：浅睡眠比例增加，深睡眠减少，能量恢复×0.7
- 压力低+信任高：深睡眠增加，能量恢复×1.3，信任≥4偶尔露腹睡觉

**活动状态最短停留时间**
- 浅睡眠：5分钟
- 深睡眠：15分钟
- 休息：3分钟
- 活跃：无限制

### 12. 主人识别机制

**Phase 1**
- 鼠标 = 主人的手
- 键盘/语音 = 主人的声音
- 猫感知"有人在操作电脑"，不区分具体是谁
- 信任等级是全局的（这台电脑 = 主人）

**Phase 2 升级方向**
- 语音识别：不同人说不同反应
- 打字节奏：识别主人习惯
- 摄像头：看到是谁（隐私需处理）

### 13. 环境感知系统

猫如何感知屏幕内外的世界：

| 感知维度 | 说明 |
|---------|------|
| 鼠标活动 | 有输入 = 主人在；长时间无输入 = 主人可能离开 |
| 全屏应用检测 | 全屏时 ambient 减少；退出全屏猫可能抬头张望 |
| 时间感知 | 晨昏性动物（早晚活跃），深夜减少活动 |
| 声音感知 | 系统声音（打字、点击）→ 感知但不反应；突然大声 → 警惕 |
| 窗口焦点 | 切换窗口 → 猫短暂抬头；长时间不操作 → 气味标记衰减加速 |

---

### 14. 梳毛系统

**不是独立系统，是其他行为的伴随行为**

| 触发场景 | 说明 |
|---------|------|
| 进食结束后 | 梳理面部（前爪舔湿→擦脸） |
| 压力释放 | 过度理毛（高压信号） |
| 狩猎/玩耍后 | 全身梳理 |
| 休息期间 | 缓慢梳理 |
| 信任的人在身边 | 放松理毛（安全感表达） |

**过度理毛（应激指标）**
- 触发条件：压力 > 70
- 表现：同一位置反复舔舐
- 后果：可能脱毛
- 干预：给猫空间，降低压力

### 15. 语音唤醒细节

**Phase 1 方案（悬浮球）**

```
1. 用户点击悬浮球
2. 猫被唤醒 → 耳朵竖起，播放"喵？"
3. 猫看向屏幕中央（主人方向）
4. 猫进入"待命状态"
5. 主人：鼠标操作 / 输入文字 / 说话
6. 猫根据输入做出反应
7. 10秒无互动 → 猫退出待命，恢复自然行为
```

**为什么不持续收音**
- 隐私问题（麦克风一直开）
- 误触发（环境音触发猫反应）
- CPU占用（语音模型后台运行耗资源）

**Phase 2 升级（真正的语音唤醒）**
- 轻量级本地唤醒词检测（如 Piper）
- 唤醒词可自定义（如"小咪"、"喵喵"）
- 唤醒后 → Whisper STT 识别命令 → 执行 + 回应

### 16. 状态持久化

**退出时保存**
- 猫的位置（x, y）
- 所有动机变量（能量、饱腹感、压力、气味标记、信任度、猎物欲望、疲劳度、体重）
- 当前状态（睡眠/休息/活跃/进食）
- 上次保存时间（用于计算离线衰减）

**启动时恢复**
- 读取保存的状态
- 根据离线时间计算衰减：
  - 能量缓慢下降
  - 饱腹感下降
  - 气味标记大幅衰减
  - 压力可能上升
- 猫醒来，恢复自然行为

#### 养猫策略生成

根据用户选择的**品种**和**愿景**，系统自动生成个性化策略：

```
用户选择：活泼好动型 + 美短
    ↓
生成策略：
- 猎物欲望阈值：70（触发频繁，更易主动狩猎）
- 每日玩耍次数建议：至少3次（消耗精力）
- 能量消耗速率：×1.2（比默认快）
- 进食节奏：少食多餐（能量补充快）
- 预期体重：标准偏下
- 压力临界值：稍高（活跃型更能承受压力）
```

```
用户选择：粘人撒娇型 + 布偶
    ↓
生成策略：
- 信任度累积速度：×1.3（更容易建立亲密感）
- 压力恢复速度：×1.2（在信任的人身边恢复更快）
- 每日抚摸需求：高（需要更多互动）
- 猎物欲望阈值：50（较低，不太主动狩猎）
- 预期体重：标准（不太爱动）
```

#### 策略参数表

| 参数 | 影响 |
|------|------|
| 猎物欲望累积速率 | 活泼型×1.3，安静型×0.7 |
| 信任度增长速率 | 粘人型×1.3，独立型×0.8 |
| 能量消耗速率 | 活泼型×1.2，安静型×0.8 |
| 每日互动需求阈值 | 粘人型高，安静型低 |
| 压力临界值 | 活泼型更高，安静型更低 |
| 进食频率偏好 | 活泼型少食多餐，安静型一日两餐 |

#### 档案持久化

猫咪档案存储在本地，与状态持久化分开：
- 档案修改后立即保存
- 品种和愿景影响系统参数，不影响核心机制
- 用户可在设置面板随时修改

## 设计原则

1. **先研究真实域，再用工程实现** — 设计系统前先用女娲框架研究真实猫的行为
2. **关系需要时间培养** — 信任等级不是即时获得的，需要2-4周正向互动
3. **真实性优先** — 不走讨好人的捷径，尽量还原真实猫咪习性
4. **MVP 先轻再重** — 先用静态主视觉验证体验，再决定是否升级到 sprite 或程序化动画
5. **可调节的详细度** — MVP简化细节，Phase 2补充完整逻辑
