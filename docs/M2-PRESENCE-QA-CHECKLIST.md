# M2 双模式存在感手工 QA 清单

## 准备

- [ ] 已执行 `npm test`
- [ ] 已执行 `npm run build`
- [ ] 已执行 `npm run verify:m2:presence`
- [ ] 已执行 `npm run verify:m1-closure`

## 工作态

- [ ] 首次启动默认进入工作态。
- [ ] 猫咪贴在主显示器工作区边缘且可见。
- [ ] 工作态猫不会横穿屏幕中央。
- [ ] 工作态仍有轻微呼吸/浮动存在感。
- [ ] 工作态非紧急照料需求不主动弹气泡。
- [ ] 工作态紧急照料需求仍可轻量提示。

## 闲置态

- [ ] 达到当前闲置阈值后进入闲置态。
- [ ] 闲置态恢复 M1 自由漫游。
- [ ] 闲置态照料提示行为与 M1 一致。
- [ ] 用户恢复活动后回到工作态并重新贴边。

## 托盘菜单

- [ ] 托盘显示当前模式。
- [ ] 托盘显示当前控制方式。
- [ ] 托盘显示当前闲置阈值。
- [ ] 可选择 `自动`。
- [ ] 可选择 `强制工作态`。
- [ ] 可选择 `强制闲置态`。
- [ ] 可选择 `10 分钟`。
- [ ] 可选择 `30 分钟`。
- [ ] 可选择 `1 小时`。
- [ ] 阈值选择重启后保留。

## 照料回归

- [ ] `Feed` 在工作态和闲置态均可用。
- [ ] `Give water` 在工作态和闲置态均可用。
- [ ] `Pet` 在工作态和闲置态均可用。
- [ ] `Food / Water / Play / Trust` 面板持续可见。
- [ ] 打开验证报告可看到 presence mode、阈值与照料事件摘要。
